54  // A function that is called in response to a RTSP command. The parameters are as follows:
55  // "rtspClient": The "RTSPClient" object on which the original command was issued.
56  // "resultCode": If zero, then the command completed successfully. If non-zero, then the command did not complete
57  // successfully, and "resultCode" indicates the error, as follows:
58  // A positive "resultCode" is a RTSP error code (for example, 404 means "not found")
59  // A negative "resultCode" indicates a socket/network error; 0-"resultCode" is the standard "errno" code.
60  // "resultString": A ('\0'-terminated) string returned along with the response, or else NULL.
61  // In particular:
62  // "resultString" for a successful "DESCRIBE" command will be the media session's SDP description.
63  // "resultString" for a successful "OPTIONS" command will be a list of allowed commands.
64  // Note that this string can be present (i.e., not NULL) even if "resultCode" is non-zero - i.e., an error message.
65  // Also, "resultString" can be NULL, even if "resultCode" is zero (e.g., if the RTSP command succeeded, but without
66  // including an appropriate result header).
67  // Note also that this string is dynamically allocated, and must be freed by the handler (or the caller)
68  // - using "delete[]".
